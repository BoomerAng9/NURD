## 3. What Changed in This Batch

### Bug found and fixed: vite import hoisted to bundle root

**Symptom:** Container crashed on boot:
```
Error [ERR_MODULE_NOT_FOUND]: Cannot find package 'vite' imported from /app/dist/index.js
```

**Root cause:** `server/vite.ts` had `import viteConfig from "../vite.config"`. esbuild followed the import to `vite.config.ts`, which has `import { defineConfig } from "vite"` at its top. esbuild bundled `vite.config.ts` inline into `dist/index.js` and **hoisted vite.config's static import of `vite` to the bundle root**. At runtime in the prod-deps Docker stage (vite is a devDep, pruned by `npm ci --omit=dev`), Node failed to resolve the hoisted `import "vite"` and crashed.

**Fix (in `server/vite.ts`):**
1. Converted all dev-only imports (`vite`, `nanoid`) to dynamic `await import(...)` inside `setupVite()` — this defers resolution past the `NODE_ENV === 'production'` early-return guard.
2. Removed `import viteConfig from "../vite.config"` entirely. Inlined the dev-relevant config (resolve.alias, root, react plugin) directly in `setupVite()`. `vite.config.ts` is still used by `vite build` at compile time; just not loaded at runtime.
3. Replaced `import path from "path"` to also use a precomputed `repoRoot` for clarity.

### Files changed
- `server/vite.ts` — re-architected to load vite/plugin-react/nanoid only in dev mode, no runtime dependency on vite.config
